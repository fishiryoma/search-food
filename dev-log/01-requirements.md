# 需求規格

## 原始需求（使用者描述，保留原文）

1. 抓取現在的位置，並串聯 Google Map
2. 搜尋步行範圍可抵達 1 公里內的範圍
3. 篩選出所有餐廳
4. 粗略分析餐廳類型
5. 讓使用者選擇餐廳價位、餐廳類型
6. 依照使用者所選的類型，再去篩選出相關的餐廳
7. 再次分析篩選出的餐廳餐點、價位，並提供選項給使用者選擇，以限縮條件
8. 依照新的條件列出 5 間餐廳
9. 點擊可直接連結打開 Google Map

## 細化後的功能規格

### 步驟 0：定位授權
- **UI**：「允許取得您的位置」按鈕
- **行為**：呼叫 `navigator.geolocation.getCurrentPosition`
- **錯誤狀態**：拒絕授權時顯示提示，引導手動輸入地址（MVP 暫不實作手動輸入）

### 步驟 1：地圖顯示 + 搜尋
- **UI**：Google Maps 元件，使用者位置 marker + 1km 半徑圓圈
- **行為**：自動呼叫 `/nearby` 端點（lat, lng, radius=1000）
- **地圖標記**：所有找到的餐廳以 marker 標示
- **簡化說明**：「步行 1km」以圓形半徑實作（非真實等時線），MVP 接受此簡化

### 步驟 2：粗篩 UI
- **UI**：兩組多選 chip
  - 餐廳類型（由 Places API 的 `types` 欄位聚合出現次數最高者）
  - 價位（Google 的 `priceLevel` 1–4，顯示為 $ / $$ / $$$ / $$$$）
- **行為**：使用者選擇後，前端即時過濾 marker 並計算候選清單

### 步驟 3：LLM 分析（第二階段）
- **觸發**：使用者點擊「深入分析」或「再幫我篩選」
- **UI**：Loading 狀態「分析餐點中…」
- **行為**：呼叫 `/analyze` 端點，傳入粗篩後的餐廳清單

### 步驟 4：細項篩選
- **UI**：LLM 回傳的標籤，做成多選 chip
  - 例：菜系（日式 / 台式 / 韓式 / 西式…）、口味（清淡 / 重口味 / 素食…）、場景（外帶 / 內用 / 快食…）
- **行為**：使用者勾選後呼叫 `/refine` 端點

### 步驟 5：結果列表
- **UI**：5 間餐廳卡片，每張顯示：名稱、類型標籤、評分、價位、距離（公尺）
- **行為**：點擊卡片 → 新分頁開啟 `https://www.google.com/maps/search/?api=1&query=PLACE_ID:<place_id>`

## 非功能需求

| 需求 | 目標值 |
|---|---|
| 首次搜尋回應時間 | < 3 秒（含 Places API） |
| LLM 分析時間 | < 5 秒 |
| Google Maps 月費 | 個人使用維持在 $200 免費額度內 |
| API Key 安全 | 伺服器端 Keys 不得出現在前端 bundle |
| 行動裝置相容 | iOS Safari / Android Chrome |

## Out of Scope（MVP 刻意不做）

- 使用者帳號與偏好儲存
- 真實步行等時線（Routes API isochrone）
- 餐廳評論詳細頁
- 多語言（只支援繁體中文）
- 離線模式
